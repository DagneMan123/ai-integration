PROBLEM IDENTIFIED:
──────────────────
When users tried to submit answers during an interview, they got:
"Interview not in progress - /api/interviews/21/submit-answer"

ROOT CAUSE:
The interview was never being created with the /interviews/start endpoint.
The flow was:
1. User clicks "Start AI Interview" → Redirects to Dashboard
2. Dashboard shows payment prompt
3. User clicks "Start Interview Now" → Navigates to /candidate/interview/{id}
4. BUT: /interviews/start was NEVER called
5. Interview doesn't exist or has wrong status
6. Submit answer fails

═══════════════════════════════════════════════════════════════════════════════

SOLUTION IMPLEMENTED:
────────────────────

FILE 1: client/src/pages/candidate/Dashboard.tsx
─────────────────────────────────────────────────
CHANGE: When user clicks "Start Interview Now" (has enough credits)

BEFORE:
  navigate(`/candidate/interview/${pendingInterviewId}`);

AFTER:
  1. Call /interviews/start with jobId and applicationId
  2. Get actual interviewId from response
  3. Navigate to /candidate/interview/{actualInterviewId}
  4. Clear localStorage

RESULT: Interview is created with IN_PROGRESS status before navigation


FILE 2: client/src/pages/PaymentSuccess.tsx
────────────────────────────────────────────
CHANGE: After successful payment verification

BEFORE:
  navigate(`/candidate/interview/${interviewId}`);

AFTER:
  1. Call /interviews/start with jobId and applicationId
  2. Get actual interviewId from response
  3. Navigate to /candidate/interview/{actualInterviewId}
  4. Clear localStorage

RESULT: Interview is created with IN_PROGRESS status after payment


═══════════════════════════════════════════════════════════════════════════════

COMPLETE FLOW NOW:
──────────────────

1. User clicks "Start AI Interview" in Interviews page
   ↓
2. Redirected to Dashboard with payment prompt
   ↓
3. User has enough credits:
   - Click "Start Interview Now"
   - Backend: POST /interviews/start → Creates interview (IN_PROGRESS)
   - Frontend: Navigate to /candidate/interview/{id}
   ↓
4. User doesn't have enough credits:
   - Click "Pay & Start Interview"
   - Redirected to Chapa payment
   - After payment: POST /interviews/start → Creates interview (IN_PROGRESS)
   - Frontend: Navigate to /candidate/interview/{id}
   ↓
5. Interview Session loads
   - Fetches interview details
   - Displays questions
   ↓
6. User submits answer
   - POST /interviews/{id}/submit-answer
   - Status check: interview.status === 'IN_PROGRESS' ✅ PASSES
   - Answer is recorded
   ↓
7. Interview completes
   - POST /interviews/{id}/complete
   - Interview marked as COMPLETED
   - Redirect to report page
