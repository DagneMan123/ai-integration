ISSUE: Payment failed with errors on Chapa payment integration

ROOT CAUSES FIXED:
  1. ✅ Webhook handler looking for wrong field (transactionRef → transactionId)
  2. ✅ Webhook handler using wrong status case (completed → COMPLETED)
  3. ✅ Missing webhook signature validation
  4. ✅ Poor error logging for debugging

FILES MODIFIED:
  • server/controllers/paymentController.js (webhook handler)
  • server/services/chapaService.js (error logging)

================================================================================
                              PAYMENT FLOW
================================================================================

1. User clicks "Subscribe Now"
   ↓
2. Frontend sends payment request
   ↓
3. Backend creates Payment record (status: PENDING)
   ↓
4. Backend calls Chapa API
   ↓
5. Chapa returns checkout URL
   ↓
6. Frontend redirects to Chapa checkout
   ↓
7. User completes payment
   ↓
8. Chapa redirects back to app
   ↓
9. Frontend verifies payment
   ↓
10. Backend updates Payment status to COMPLETED ✅
   ↓
11. Chapa sends webhook notification
   ↓
12. Backend updates Payment status to COMPLETED ✅
   ↓
13. User sees success message

                        TROUBLESHOOTING
================================================================================

Issue: "Payment not found" during verification
  → Check transactionId format: TX-timestamp-random
  → Verify Payment record exists in database

Issue: "Chapa secret key is not configured"
  → Check server/.env has CHAPA_SECRET_KEY
  → Restart server after updating .env

Issue: "Invalid amount" error
  → Ensure amount > 0
  → Amount should be in ETB (Ethiopian Birr)

Issue: Webhook not updating payment
  → Check webhook uses transactionId (not transactionRef)
  → Verify webhook signature validation
  → Check server logs for webhook errors
