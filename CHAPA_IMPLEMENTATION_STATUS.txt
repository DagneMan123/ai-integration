================================================================================
                            PAYMENT FLOW
================================================================================

1. User opens BillingSidebar
   ↓
2. Sees wallet balance (e.g., 5 credits)
   ↓
3. Selects credit bundle (e.g., 10 credits for 50 ETB)
   ↓
4. Clicks "Top Up" button
   ↓
5. Frontend generates unique tx_ref
   ↓
6. Frontend sends payment initialization request:
   POST /api/payments/initialize
   {
     "bundleId": 1,
     "amount": 25,
     "creditAmount": 5,
     "txRef": "simuai_1234567890_abc123",
     "metadata": { ... }
   }
   ↓
7. Backend creates PENDING payment record
   ↓
8. Backend generates Chapa checkout URL
   ↓
9. Backend returns response:
   {
     "success": true,
     "data": {
       "checkout_url": "https://api.chapa.co/v1/hosted/pay/...",
       "txRef": "simuai_1234567890_abc123",
       "amount": 25,
       "creditAmount": 5
     }
   }
   ↓
10. Frontend redirects to Chapa payment gateway
    ↓
11. User selects payment method (Telebirr/CBEBirr/Bank)
    ↓
12. User completes payment
    ↓
13. Chapa sends webhook callback:
    POST /api/payments/webhook
    {
      "tx_ref": "simuai_1234567890_abc123",
      "status": "completed",
      "amount": 25,
      "reference": "chapa_ref_123"
    }
    ↓
14. Backend verifies HMAC-SHA256 signature
    ↓
15. Backend processes payment atomically:
    - Update Payment status to COMPLETED
    - Get or create Wallet
    - Increment wallet balance (5 + 10 = 15 credits)
    - Log transaction
    ↓
16. User redirected to success page
    ↓
17. Wallet balance updated in real-time
    ↓
18. User can now start interviews with new credits

================================================================================
                            API ENDPOINTS
================================================================================

PAYMENT ENDPOINTS:
  POST   /api/payments/initialize      - Start payment (auth required)
  POST   /api/payments/webhook         - Chapa callback (signature validation)
  GET    /api/payments/bundles         - Get credit bundles (no auth)
  GET    /api/payments/history         - Payment history (auth required)
  GET    /api/payments/analytics       - Financial analytics (auth required)
  GET    /api/payments/export          - Export as CSV (auth required)

WALLET ENDPOINTS:
  GET    /api/wallet/balance           - Current balance (auth required)
  GET    /api/wallet/transactions      - Transaction history (auth required)
  GET    /api/wallet/check-credits     - Check if sufficient credits (auth required)
