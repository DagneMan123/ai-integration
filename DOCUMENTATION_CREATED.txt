⚠️ WHAT REQUIRES USER ACTION
═════════════════════════════════════════════════════════════════════════════

1. Start PostgreSQL Database
   - Must be running on port 5432
   - Start via Windows Services
   - Or use command line

2. Start Backend Server
   - Command: cd server && npm run dev
   - Port: 5000
   - Must start AFTER PostgreSQL

3. Start Frontend Client
   - Command: cd client && npm run dev
   - Port: 3000
   - Must start AFTER backend server


🚀 QUICK START COMMANDS
═════════════════════════════════════════════════════════════════════════════

Start PostgreSQL:
  services.msc → postgresql-x64-15 → Start

Start Backend (Terminal 1):
  cd server
  npm run dev

Start Frontend (Terminal 2):
  cd client
  npm run dev

Run Diagnostics:
  node diagnose-system.js

Access Application:
  http://localhost:3000

📞 TROUBLESHOOTING QUICK REFERENCE
═════════════════════════════════════════════════════════════════════════════

Error: "Can't reach database server"
→ Start PostgreSQL service

Error: "Route.get() requires callback"
→ Restart backend server (already fixed)

Error: "Loading chunk failed (timeout)"
→ Clear client build cache

Error: "Port already in use"
→ Kill process on that port

Error: "npm: command not found"
→ Install Node.js
