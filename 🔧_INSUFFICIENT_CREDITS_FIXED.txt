1. ✅ DECIMAL TYPE COMPARISON BUG
   File: server/controllers/interviewController.js
   
   Problem: Wallet balance (Decimal type) wasn't being converted to number
   before comparison, causing comparison to fail.
   
   Fix: Added parseFloat() to convert Decimal to number:
   const balanceAmount = parseFloat(wallet.balance);
   if (balanceAmount < 1) { ... }

2. ✅ MISSING CREDIT BUNDLES
   File: server/prisma/seed.js
   
   Problem: Database seed didn't create any credit bundles, so users
   couldn't purchase credits.
   
   Fix: Added 3 credit bundles to seed:
   - Starter Pack: 5 credits for 25 ETB
   - Professional Pack: 10 credits for 45 ETB
   - Enterprise Pack: 25 credits for 100 ETB

Interview Cost:
  1 Interview = 1 Credit

Payment Flow:
  1. User clicks "Pay & Start Interview"
  2. System checks wallet balance
  3. If balance < 1, shows payment modal
  4. User selects credit bundle
  5. Payment processed via Chapa
  6. Credits added to wallet
  7. Interview starts

Credit Deduction:
  1. User starts interview
  2. System checks balance (must be ≥ 1)
  3. Deducts 1 credit from wallet
  4. Interview session starts
  5. Transaction logged
