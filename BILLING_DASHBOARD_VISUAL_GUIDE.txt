┌─────────────────────────────────────────────────────────────────────────────┐
│                    USER INTERACTION FLOW                                    │
└─────────────────────────────────────────────────────────────────────────────┘

    Scenario 1: Top Up Credits from Dashboard
    ┌─────────────────────────────────────────────────────────────┐
    │ 1. User on Candidate Dashboard                              │
    │    ↓                                                         │
    │ 2. Clicks "Billing" button (top right)                      │
    │    ↓                                                         │
    │ 3. BillingSidebar opens from right                          │
    │    ↓                                                         │
    │ 4. Sidebar shows wallet balance (10 credits)                │
    │    ↓                                                         │
    │ 5. User selects bundle (e.g., 10 credits for 50 ETB)        │
    │    ↓                                                         │
    │ 6. User clicks "Top Up" button                              │
    │    ↓                                                         │
    │ 7. Payment initialization request sent                      │
    │    ↓                                                         │
    │ 8. Redirected to Chapa payment gateway                      │
    │    ↓                                                         │
    │ 9. User completes payment                                   │
    │    ↓                                                         │
    │ 10. Webhook callback received                               │
    │    ↓                                                         │
    │ 11. Wallet updated (10 + 10 = 20 credits)                   │
    │    ↓                                                         │
    │ 12. User redirected to success page                         │
    │    ↓                                                         │
    │ 13. Sidebar refreshes and shows new balance                 │
    └─────────────────────────────────────────────────────────────┘

    Scenario 2: View Payment History
