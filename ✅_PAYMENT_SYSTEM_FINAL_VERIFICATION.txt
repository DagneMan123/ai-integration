COMPLETE PAYMENT TO INTERVIEW FLOW
─────────────────────────────────────────────────────────────────────────────────

User clicks "Start AI Interview"
    ↓
Dashboard shows payment modal
    ↓
User clicks "Pay & Start Interview"
    ↓
Frontend calls: POST /api/payments/initialize
    ↓
Backend creates PENDING payment record
    ↓
Backend generates Chapa checkout URL
    ↓
Frontend redirects to Chapa
    ↓
User enters payment details MANUALLY (4200000000000000, 12/25, 123)
    ↓
User clicks "Pay" on Chapa
    ↓
Chapa processes payment
    ↓
Chapa redirects to: /payment/success?tx_ref={txRef}
    ↓
Frontend calls: GET /api/payments/verify/{txRef}
    ↓
Backend verifies with Chapa API
    ↓
Backend updates payment to COMPLETED
    ↓
Backend updates wallet with 1 credit
    ↓
Frontend shows success page with confetti
    ↓
5-second countdown
    ↓
Auto-redirect to: /candidate/interview/{interviewId}?paymentVerified=true
    ↓
Interview page loads
    ↓
User can answer questions

PAYMENT FLOW ENDPOINTS
─────────────────────────────────────────────────────────────────────────────────

POST /api/payments/initialize
  ✅ Requires: JWT token, amount, creditAmount
  ✅ Returns: txRef, checkout_url, amount, creditAmount
  ✅ Creates: PENDING payment record
  ✅ Generates: Chapa checkout URL

GET /api/payments/verify/:txRef
  ✅ Requires: JWT token, txRef parameter
  ✅ Returns: success, txRef, amount, status, creditAmount, paidAt
  ✅ Verifies: Payment with Chapa API
  ✅ Updates: Payment status to COMPLETED
  ✅ Updates: Wallet with credits

GET /api/payments/history
  ✅ Requires: JWT token
  ✅ Returns: Payment history with pagination
  ✅ Supports: Filtering by status, date range
  ✅ Supports: Sorting by date, amount

GET /api/payments/analytics
  ✅ Requires: JWT token
  ✅ Returns: Total spent, successful transactions, failed transactions

POST /api/payments/webhook
  ✅ No auth required (signature validation instead)
  ✅ Verifies: HMAC-SHA256 signature
  ✅ Processes: Payment completion
  ✅ Updates: Payment status and wallet
